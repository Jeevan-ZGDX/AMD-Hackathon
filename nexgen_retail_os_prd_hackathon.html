06 — Hackathon Roadmap

From 0 to demo
in 36 hours.

Hour 0–6 · Phase 1
Core Data Pipeline + Auth
Set up Next.js 14 app, Supabase for session data, PostgreSQL with pgvector. Ingest 5K dummy SKUs, 10K synthetic user profiles with behavioral vectors. Build JWT auth + onboarding.
Hour 6–14 · Phase 2
Pre-Cognitive Engine + Heatmap
Build behavioral event tracker (scroll, hover, dwell). Train lightweight intent classifier (Python + scikit-learn, served via FastAPI). Build Retailer Heatmap using D3.js + WebSocket live updates.
Hour 14–22 · Phase 3
Zero-Click Checkout + Pricing Engine
Build cart auto-assembly logic. Push notification via Expo (PWA fallback). Autonomous pricing engine with mock competitor API + margin floor config. Payment stub via Razorpay sandbox.
Hour 22–36 · Phase 4
Polish, Demo Prep + Pitch Deck
Aggressive scroll animations via GSAP ScrollTrigger. Emotion mode toggle UI. Live demo walkthrough script. Retailer dashboard with KPI cards. Final QA + deploy to Vercel.

08 — Antigravity Prompt

The brief.
Verbatim.

Use this as your hackathon submission prompt for Claude / GPT-4o / Gemini to auto-generate boilerplate.

SYSTEM CONTEXT: You are a senior full-stack AI engineer with 15+ years of experience, specializing in retail-tech, behavioral AI, and real-time systems. You write production-grade code with zero excuses.

PROJECT: NexGen Retail OS — an AI-powered retail operating system that predicts what shoppers want before they search, builds their cart automatically, and gives retailers a real-time demand heatmap.

OBJECTIVE: Build a full-stack hackathon demo with 4 modules: (1) Pre-Cognitive Engine that reads scroll/hover/dwell signals and predicts shopping intent using a lightweight ML model, (2) Zero-Click Checkout that assembles a cart at 92% confidence and sends a one-tap push notification, (3) Live Demand Heatmap for retailers showing SKU demand by geography updated every 90s via WebSocket, (4) Autonomous Pricing Engine with competitor price scraping simulation and margin floor logic.

TECH STACK: Next.js 14 + Tailwind + GSAP for frontend · FastAPI + scikit-learn + pgvector for ML API · Supabase + PostgreSQL + Redis for data · Vercel deploy · All dummy data via Faker.js and Mockaroo. NO external paid APIs — everything runs on sandbox/free tier.

UI REQUIREMENT: Dark, aggressive editorial design. Full-page scroll sections. Scroll-triggered animations via GSAP ScrollTrigger on every section entry. Retailer dashboard with live WebSocket KPI cards. Shopper view with pre-built cart reveal animation. Mobile responsive.

DELIVERABLE: Complete working repo with README, .env.example, seeding scripts for 5K SKUs and 10K user profiles, and a 3-minute demo walkthrough script. Start with the folder structure, then generate each module sequentially. Do not skip error handling. Do not use placeholder comments — write real code.
